\moduledescription{mod.cryptsystem.keymgmt.desc}

The module \tds{mod.cryptsystem.keymgmt} of subsystem
\tdslink{sub.cryptsystem} provides functionality to manage cryptographic keys.

\moduleprocesses{prc.cryptsystem.keymgmt}

\moduleprocess{prc.cryptsystem.keymgmt.genkey}{Creating Keys}

Ephemeral Diffie-Hellman key pairs are created by reading random numbers from
\tdslink[fq]{mod.cryptsystem.rng}. The random data is wrapped into the
appropriate data structures.

\implementedsfr{\sfrlink{fcs_ckm.1}}

\moduleprocess{prc.cryptsystem.keymgmt.destroykey}{Destroying keys}

Keys are destroyed by overwriting the data with pseudo-random numbers. Module
\tdslink[fq]{mod.selfprotect.memory} is used for securely erasing the memory.

\implementedsfr{\sfrlink{fcs_ckm.4}}

\moduleinterfaces{int.cryptsystem.keymgmt}

\moduleinterfaceprovided{int.cryptsystem.keymgmt.createkey}

This interface is used to create a key pair (see
\sectref{prc.cryptsystem.keymgmt.genkey}).

\moduleinterfaceprovided{int.cryptsystem.keymgmt.destroykey}

This interface is used to destroy a key (see \sectref{prc.cryptsystem.keymgmt.destroykey}).

\moduleinterfacerequired{int.cryptsystem.rng.creaternd}

The interface \tdslink[fq]{int.selfprotect.memory.cleanup} is required for
generating random numbers (see \sectref{prc.cryptsystem.keymgmt.genkey})

\moduleinterfacerequired{int.selfprotect.memory.cleanup}

The interface \tdslink[fq]{int.selfprotect.memory.cleanup} is required for
securely erasing the memory during key deletion (see
\sectref{prc.cryptsystem.keymgmt.destroykey}).
